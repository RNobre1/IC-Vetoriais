# Sessão 2026-05-06 — Bump de versões antes do Dia 2

## TL;DR
Antes de começar o Dia 2 (pipeline + seeders), fizemos um bump das 3 imagens Docker e dos 5 clientes Python pareados, do snapshot de outubro/2024 da ADR original para um snapshot de maio/2026 contemporâneo ao início dos experimentos. Smoke test segue **6/6 verde** com as versões novas. Toda a documentação (relatório parcial, plano da Etapa 2, README de `code/`, ADR de isolamento UI) foi atualizada para refletir o bump.

**Próximo passo concreto:** Etapa 2 / Dia 2 — `pipeline/ms_marco_loader.py` (TDD) + `pipeline/embeddings.py` + os 3 seeders. Plano detalhado em [`../../docs/tasks/etapa-2-preparacao-ambiente.md`](../../docs/tasks/etapa-2-preparacao-ambiente.md).

## O que foi feito nesta sessão

### Bump de infraestrutura
- **Imagens Docker** (em `code/docker-compose.yml`):
  - `pgvector/pgvector:pg16` → `pgvector/pgvector:0.8.2-pg18-bookworm` (Postgres 18.3 + pgvector 0.8.2)
  - `qdrant/qdrant:v1.12.0` → `qdrant/qdrant:v1.17.1`
  - `semitechnologies/weaviate:1.27.0` → `semitechnologies/weaviate:1.37.2`
- **Clientes Python** (em `code/requirements.txt` e `requirements.lock`):
  - `psycopg[binary,pool]` 3.2.13 → 3.3.4 (compat pg18)
  - `pgvector` (adapter Python) 0.3.6 → 0.4.2
  - `qdrant-client` 1.12.1 → 1.17.1 (pareado com server)
  - `weaviate-client` 4.9.6 → 4.21.0
  - `httpx` 0.27.0 → 0.28.1 (desbloqueado pelo bump do weaviate-client)
- **Volume mount do Postgres** ajustado de `/var/lib/postgresql/data` para `/var/lib/postgresql` (breaking change da imagem pg18+).
- **Smoke test** ajustado para a API nova:
  - Qdrant: `client.search()` → `client.query_points(...).points` (search foi removido na 1.17).
  - Weaviate: `vectorizer_config=Configure.Vectorizer.none()` + `vector_index_config=` → `vector_config=Configure.Vectors.self_provided(vector_index_config=...)` (consolidação da API em v4).

- **Weaviate**: descobrimos que `console.weaviate.cloud` **não conecta a self-hosted** (só clusters WCS hospedados). Lição registrada em [[../lições/2026-05-05-erro-recomendacao-weaviate-cloud-console]]. Solução: **Verba** isolado.

### Migração docx → LaTeX
- Bolsista migrou manualmente o `relatorio_parcial.docx` para projeto LaTeX modular em `docx/relatorio_parcial/` (template `article` + `helvet` + `abntex2cite[alf]`, 8 seções modulares + `refs.bib` com 13 referências, README documentando convenções).
- Instalado `texlive-scheme-medium` + `texlive-abntex2` + `texlive-collection-langportuguese` + `texlive-collection-fontsrecommended` + `biber` no Fedora.
- Compilação local validada: `latexmk -pdf main.tex` → `main.pdf` 16 páginas, bibliografia ABNT resolvida. 1 warning de overfull hbox no cronograma (linha 13-14 de `08-cronograma.tex`) — apenas visual.
- Adicionados ao projeto LaTeX: `.gitignore` (artefatos `.aux/.bbl/.log/main.pdf`) e `Makefile` (`pdf`, `watch`, `clean`, `distclean`).
- ADR registrada: [[../decisões/2026-05-05-migracao-relatorio-para-latex]].
- `CLAUDE.md` raiz e memória global `feedback_no_docx_edits` atualizadas: `.tex` é editável normalmente; regra de "não editar `.docx`" agora aplica só a binários legados (e.g. `OFC- Planejamento...`).

### Isolamento UI vs benchmark
- Criado `code/docker-compose.ui.yml` com serviço `verba` (imagem `semitechnologies/verba:latest`) apontando para `http://weaviate:8080` via DNS interna do compose.
- Estendido `code/Makefile` com `ui-up`, `ui-down`, `ui-logs` (combinando `-f docker-compose.yml -f docker-compose.ui.yml`).
- Estendido `code/.env.example` com `VERBA_PORT=8000` e chaves opcionais de LLM (comentadas).
- ADR registrada: [[../decisões/2026-05-05-isolamento-ui-vs-benchmark]] — fundamentada em Aumüller et al. (2020), ANN-Benchmarks. Regra: derrubar Verba antes de qualquer benchmark.
- Verba subiu: `ic-verba` Up, HTTP 200 em http://localhost:8000.
